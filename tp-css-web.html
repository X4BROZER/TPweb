EX 01

Sélecteurs & Spécificité

20 min
1.1 Tous les liens dans la navigation
+
.main-nav a /* Descendant : tout <a> dans .main-nav */
1.2 Uniquement le lien actif
+
.main-nav a.active
Cible un <a> qui possède simultanément la classe active, à l'intérieur de .main-nav.
1.3 L'article avec la classe featured
+
article.post.featured  /* ou simplement .post.featured */
Chaîner deux classes sans espace signifie que l'élément doit avoir les deux.
1.4 Le premier paragraphe de chaque article
+
.post p:first-of-type
:first-of-type cible le premier enfant de type <p> dans chaque .post.
1.5 Les éléments <li> pairs
+
li:nth-child(even)  /* ou li:nth-child(2n) */
1.6 Classement par spécificité croissante
+
p (0,0,1)
1 élément
.text (0,1,0)
1 classe
p.text (0,1,1)
classe + élément
#main (1,0,0)
1 ID
#main .text p (1,1,1)
ID + classe + élément
Égalité de spécificité : la règle déclarée en dernier dans le CSS l'emporte (effet cascade).
Éviter !important : il court-circuite la cascade et oblige à multiplier les !important pour surcharger, rendant le code très difficile à maintenir.

+

2.2 Largeur totale en border-box
+
Avec border-box, la propriété width: 300px inclut padding et border. 
/* Largeur visible = 300px (toujours) */
/* Contenu réel = 300 - 40 - 10 = 250px */
Largeur totale visible = 300px
2.3 Reset CSS universel pour box-sizing
+

  box-sizing: border-box;
}
À placer en tout début de feuille de style. Garantit un comportement prévisible sur tous les éléments.

EX 05

Responsive Design

25 min
5.1 Media queries Mobile First (tablette + desktop)
+
/* ── Base : Mobile (< 768px) ── */
.container { width: 100%; padding: 15px; }
.nav-menu  { display: flex; flex-direction: column; gap: 5px; }
.content  { display: block; }
.sidebar  { display: none; }

/* ── Tablette (≥ 768px) ── */
@media (min-width: 768px) {
  .container { max-width: 720px; margin: 0 auto; }
  .nav-menu  { flex-direction: row; justify-content: center; }
  .sidebar  { display: block; }
}

/* ── Desktop (≥ 1024px) ── */
@media (min-width: 1024px) {
  .container { max-width: 960px; }
  .content   {
    display: grid;
    grid-template-columns: 1fr 300px;
    gap: 2rem;
  }
}
Mobile First = on écrit d'abord les styles pour petit écran, puis on enrichit avec min-width. C'est l'opposé de max-width (Desktop First).

QUIZ

Révision — Réponses

1. Quel sélecteur a la spécificité la plus haute ?
.header .nav a #header a ✓ header nav a a.active
#header a — un ID compte pour (1,0,0), bien au-dessus des classes (0,1,0) et éléments (0,0,1).
2. Avec box-sizing: border-box, un élément width: 200px et padding: 20px a une largeur totale de :
240px 200px ✓ 220px 160px
border-box inclut padding et border dans les 200px déclarés. La largeur totale reste donc 200px.
3. Pour centrer un élément verticalement ET horizontalement avec Flexbox :
justify-content: center align-items: center justify-content: center ET align-items: center ✓ text-align: center
Les deux propriétés sont nécessaires : justify-content pour l'axe principal (horizontal), align-items pour l'axe transversal.
4. La valeur 1fr en CSS Grid signifie :
1 pixel 1 fraction de l'espace disponible ✓ 1 pourcentage 1 rem
fr = fraction unit. 1fr prend une fraction proportionnelle de l'espace libre restant dans le conteneur grid.
5. L'approche "Mobile First" utilise :
max-width dans les media queries min-width dans les media queries ✓ device-width orientation
Mobile First = on part du plus petit écran et on élargit progressivement avec min-width.
